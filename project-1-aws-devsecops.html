BACKGROUND

01 기획 배경

최근 클라우드 전환이 가속화되면서 초기 클라우드 설정 오류로 인한 대규모 데이터 유출 사례가 빈번하게 발생하고 있습니다.
해당 문제에 대한 원인을 분석한 결과 세 가지 문제로 좁혀졌습니다.


1. 배포 전 IaC 검증 부족 — 초기 설정이 미흡한 상태로 인프라가 배포되는 문제
2. 보안 도구 결과의 파편화 — 여러 보안 도구의 결과가 흩어져 있어 문제 발생 지점을 신속히 파악하기 어려운 문제
3. CI/CD 로그 가시성 부족 — 비정형적인 로그로 사고 원인 분석에 제약이 생기는 문제


그래서 배포 흐름을 통합적으로 관리하고, 보안 로그 분석을 AI 기반으로 자동화하여 실무자 부담을 줄이는
AWS DevSecOps 플랫폼이 필요하다고 판단해 프로젝트를 기획했습니다.

KEY FEATURES

04 핵심 기능

1. GitHub Actions 실시간 연동 + AI 로그 분석

GitHub Actions 탭에서 전체 Action 로그를 실시간으로 확인하고, 각 단계를 한글로 요약해 보여줍니다.
문제 발생 시에는 로그 데이터를 AI가 분석해 해결 방안을 제시합니다.

AI 로그 분석 흐름
Job 정보 백엔드 전송 패턴 매칭으로 1차 초안 작성 로그 / Terraform 코드 결합 Gemini API 호출 DevOps 엔지니어 역할 프롬프트 적용 핵심 요약 및 해결 가이드 출력

인증을 위해 Organization 내에 Git App을 생성하고 최소 권한만 부여한 뒤,
installation id·토큰을 발급받아 백엔드 서버가 GitHub Actions REST API 권한을 획득하도록 구성했습니다.
이후 15초 주기 polling으로 workflow run/job/step 상태를 실시간 유지합니다.


2. 보안 정책 자동화 (PDF → Checkov Policy)

KISA 가이드 등 PDF 정책 문서를 업로드하면 AI가 분석해 Checkov에 바로 적용 가능한 YAML 정책으로 자동 변환합니다.

PDF 정책 자동화 흐름
PDF 업로드 Frontend Base64 인코딩 Backend 디코딩 / PDF 파싱 필수 정보 추출 Gemini API 전달 Checkov YAML 정책 생성 리소스 타입 / YAML 2차 검증 정책 활성화 후 Git Push 반영


3. 관제 & 침해사고 대응 탭 — AI 요약 보고서

수집된 로그를 관제(서비스 상태·가용성)와 침해사고 대응(원인 분석) 두 축으로 나누어 시각화하고,
각각에서 AI가 실제 로그 데이터에 근거한 운영 요약 보고서를 자동 생성합니다.

05 문제 해결 사례 — Prometheus 수집 실패

01
문제

ECS에 Prometheus를 배포하고 API 서버 타겟은 정상 인식되었으나,
실제 수집 단계에서 404 Not Found 에러가 발생하며 모니터링이 중단됨

02
원인

Prometheus는 기본적으로 /metrics 경로에서 데이터를 수집하도록 설계되어 있지만,
자체 API 서버(Node)는 5000번 포트의 /api/metrics 경로를 사용하고 있어 통신 규격이 불일치

03
해결

prometheus.yml.tpl 템플릿 내 metrics 경로를 /api/metrics로 명시적으로 지정
→ 표준 경로가 아닌 서비스 맞춤형 엔드포인트를 강제 인식하도록 수정

04
결과

지정된 경로에서 정상적으로 metrics를 확보, 대시보드 시각화까지 정상 동작 확인

06 성과 및 강점

1. Gemini 기반 로그 분석으로 운영 효율성 강화 수백 줄 규모의 Actions 로그를 핵심 요약과 해결 가이드로 자동 정리해 원인 파악 시간 감소
2. PDF 가이드 → Checkov 정책 자동 변환 시스템 2026 주요정보통신기반시설 가이드 PDF를 CheckOV에 적용 가능한 YAML 정책으로 변환해 파이프라인에 반영
3. CI/CD 전 구간 단일 대시보드 Push → 스캔(Gitleaks·Trivy·Checkov) → Terraform 코드 기반 인프라 생성 → 도커 이미지 배포 → ECR/ECS →
관제(CloudWatch·GuardDuty·Prometheus)까지 한 화면에서 추적

기획 목표(클라우드 보안 AWS 인프라 구축 · 보안 CI/CD 파이프라인 구축 · AI로 기존 보안 탐지 한계 극복)를
모두 실제 구현으로 달성했습니다.

AWS 보안 서비스 기반 안전한 인프라 구축, Trivy·Checkov 등 오픈소스 통합 보안 파이프라인, LLM 기반 정책 자동 변환까지
